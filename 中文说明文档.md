# 人治 vs AI治理 仿真系统中文说明文档

本项目是一个用于个人探索的社会治理推演框架。它通过“相同初始条件 + 不同治理函数”的方式，对比人治与AI治理在长期演化中的差异。

---

## 1. 项目目标与核心思想

### 1.1 为什么这样设计

现实世界中的治理结果，往往同时受到历史条件、资源禀赋、人口结构、技术水平等多因素影响。若直接比较两种治理模式，很容易混入“初始条件不同”造成的偏差。

本项目采取**控制变量思路**：

- 初始个体分布一致
- 初始资源一致
- 网络结构一致
- 随机冲击机制一致
- 仅替换治理机制（Human / AI）

这样更接近“同一起跑线下，制度差异本身带来的后果”。

### 1.2 你可以用它回答的问题

- AI治理是否一定更公平？
- 人治是否一定更腐败、低效？
- 高冲击环境下，谁更有韧性？
- 是否存在“先人治后AI”的更优切换窗口？

## 2. 模型结构

模型分两层：

1. **Agent层（个体层）**：每个人有状态变量，会生产、互动、被治理、再反馈。
2. **Institution层（制度层）**：治理函数定义“规则怎么执行”。

每个Tick（时间步）按固定顺序演化：

1. 生产
2. 网络互动
3. 治理执行
4. 税收再分配
5. 满意度更新
6. 制度合法性更新
7. 指标记录

---

## 3. Agent状态变量与现实含义

每个个体有如下属性：

- `E`（经济资本）：财富水平，影响生活质量、抗风险能力。
- `S`（社会资本）：关系网络与影响力，影响资源获取和晋升机会。
- `H`（人力资本）：技能与能力，影响生产效率与制度适配能力。
- `M`（道德取向）：利己/利他倾向，影响合作、腐败和制度摩擦。
- `I`（信息透明度）：对制度运行与社会状态的知情程度。
- `U`（满意度）：主观幸福感与系统体验的综合表达。
- `P`（权力值）：体制内影响公共决策或资源分配的能力。

这些变量会被治理方式、冲击事件、互动关系持续改变。

## 4. 两种治理机制的建模逻辑

### 4.1 人治（Human Governance）

主要特征：

- 权力更容易形成精英层聚集
- 决策可能受人情、关系和执行噪声干扰
- 腐败风险随监督不足和道德风险上升
- 晋升同时看能力与关系
- 在危机中具备一定临机弹性（本模型用`adaptive_bonus_in_crisis`表示）

现实映射：

- 优点：灵活、可变通、灾时可临场应对
- 风险：寻租、裙带、执行波动

### 4.2 AI治理（AI Governance）

主要特征：

- 规则执行一致性更高
- 传统腐败更低，但存在“规则套利”（高能力者钻规则空子）
- 信息透明高，但可能引发信息过载疲劳
- 晋升更偏能力导向
- 对偏离主流价值者施加同质化压力

为避免“AI天然无敌”的不现实设定，模型加入三类摩擦：

- `objective_mismatch`：目标函数与真实福祉错配
- `perception_lag`：系统感知/反馈存在延迟
- `protocol_delay`：危机时流程响应不够灵活

现实映射：

- 优点：一致、规模化、抗传统腐败
- 风险：目标偏置、刚性、自由空间压缩

## 5. 指标体系与“实际意义”解释

当前系统会输出时序CSV和聚合统计。以下是核心指标及其现实含义：

### 5.1 公平类

- `gini_e`（财富基尼）
  - 含义：结果公平程度。
  - 越低越公平；越高表示财富集中。

### 5.2 幸福与体验类

- `mean_u`（平均满意度）
  - 含义：整体主观生活体验。
  - 反映制度“体感”而非单纯产出。

- `median_u`（中位满意度）
  - 含义：典型个体体验。
  - 可避免均值被极端值拉偏。

### 5.3 自由与多样性类

- `std_m`（道德取向标准差）
  - 含义：价值多样性。
  - 越高表示群体异质性更强。

- `behavior_entropy`（行为熵代理）
  - 含义：行为/价值分布复杂度。
  - 越高说明系统中选择结构更丰富。

### 5.4 腐败与治理质量类

- `corruption_rate`（腐败损失率）
  - 含义：生产财富中被规则外损耗的比例。
  - 越低越好，反映制度执行清洁度。

- `legitimacy`（制度合法性存量）
  - 含义：系统被接受程度与稳定基础。
  - 综合受满意度、公平、透明、方差与强制程度影响。

### 5.5 效率与能力类

- `efficiency`（人均效率）
  - 含义：单位人口的产出能力。
  - 可用于比较“治理效率”与“福利分配”之间的权衡。

### 5.6 信息结构类

- `info_mean`（信息透明均值）
  - 含义：个体平均知情度。
  - 不等于“理解能力”，需结合满意度分析信息过载影响。

- `info_var`（信息透明方差）
  - 含义：信息是否均匀可达。
  - 方差高可能意味着信息壁垒或不对称。

### 5.7 结构状态类

- `mean_p`（平均权力值）
  - 含义：整体权力更新水平。
  - 需结合分布看是否权力集中。

- `shock`（当轮是否冲击）
  - 含义：是否发生外部扰动，便于观察恢复曲线。

## 6. 如何解读“好坏”不是单一答案

治理比较不是单指标比赛。常见冲突：

- 高效率不一定高幸福
- 高透明不一定高满意
- 高公平可能伴随动力下降
- 高稳定可能压缩自由空间

建议采用“多目标分析”而不是“唯一冠军”。

可用三类结论框架：

1. **稳态优劣**：长期平均谁更优。
2. **风险优劣**：极端冲击下谁更稳。
3. **价值优劣**：在你偏好的价值函数下谁更优（效率优先/公平优先/自由优先）。

## 7. 输出文件说明（集中管理）

### 7.1 单次运行目录

每次运行会生成一个时间戳目录，例如：

- `outputs/human_seed_11_20260504_191520/`
- `outputs/ai_seed_11_20260504_191520/`

目录内文件：

- `metrics_timeseries.csv`：每个Tick的指标序列
- `summary.json`：后50轮统计与配置快照
- `report.html`：单次中文可视化报告

### 7.2 根目录综合报告（唯一文件）

- `outputs/aggregate_report.html`

特性：

- 自动扫描当前有效运行目录
- 展示最近3次、最近10次、全部运行的重叠曲线
- 提供平均值与极值区间
- 同名覆盖（始终一个文件）
- 删除某些运行目录后，重新生成即同步更新

## 8. 常用命令

### 8.1 跑单体制

```bash
python main.py --mode single --governance human --ticks 400 --agents 400 --seed 42 --out outputs
python main.py --mode single --governance ai --ticks 400 --agents 400 --seed 42 --out outputs
```

### 8.2 跑成对对照（推荐）

```bash
python main.py --mode pair --ticks 400 --agents 400 --seed 42 --out outputs
```

### 8.3 只重建综合报告

```bash
python main.py --mode aggregate --out outputs
```

## 9. 参数调优建议（实践向）

优先先调这几项：

- `shock_probability`：看韧性曲线变化
- `oversight`：观察人治腐败与效率拐点
- `conformity_penalty`：观察AI治理下自由-稳定权衡
- `tax_rate` / `redistribution_rate`：观察公平-效率平衡

建议每组参数至少跑多seed后再下结论。

## 10. 当前实现边界（你应知道）

这是“最小可运行骨架”，不是现实社会的完整数字孪生：

- 网络互动规则仍是简化版
- 机会公平指标尚未完全独立输出
- 韧性半衰期可进一步自动化计算
- 还没有批量统计显著性检验（如置信区间带）

这不影响你做探索，但要避免“过度外推”。
